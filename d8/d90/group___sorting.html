◆ Callback

template<class T >
using CPVector::Sorting::Callback = typedef Result(*)(const T&, const T&)

Template callback type.

Used in CPVector::vector<T> in order to reference a Sorting::Callback. A Sorting::Callback stores a pointer to a function that follows the required convention.

The sorting function definition must recive two contstant references to a template type object, and must return a CPVector::Sorting::Result containing CPVector::Sorting::Equal, CPVector::Sorting::Swap, or CPVector::Sorting::Ignore.

Check CPVector::Sorting::Ascending and CPVector::Sorting::Descending as a reference.

◆ Result

typedef int8_t CPVector::Sorting::Result

return type for comparing functions

Any Result that is not Swap, Equal, or Ignore, will have no efect on the sorting functionality of the vector.

◆ Ignore

constexpr Result CPVector::Sorting::Ignore = -1
staticconstexpr

Used to indicate that the objects being compared should not be swapped.

An object must be swapped if the element being compared should go after the pivot. For example, the function CPVector::Sort::Ascending() returns ignore whenever Element > Pivot, hence Element goes after Pivot.

Examples
/examples/SortingArray.cpp.
